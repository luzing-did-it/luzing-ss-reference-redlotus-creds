FIND ACTIVE .MINECRAFT FOLDER
  Method 1 (In-game):
    Options > Resource Packs > Open Pack Folder
    Navigate up one level = root game folder
  Method 2 (Memory):
    System Informer > javaw.exe > Memory > Strings
    Filter contains: \mods or \versions
    Copy the full path shown

MODS FOLDER TIMESTAMP
  What: Check tampering via folder modification date
  How: Right-click mods folder > Properties
  RED FLAG: "Date modified" is during session/SS = tampering

MOD ANALYZER (Yarps)
  powershell -ExecutionPolicy Bypass -Command "Invoke-Expression (Invoke-RestMethod https://raw.githubusercontent.com/YarpLetapStan/PowershellScripts/refs/heads/main/YarpsModAnalyzer6.0.ps1)"

LUYTEN DECOMPILATION
  What: View .jar source code
  How: Drag .jar into Luyten
  LOOK FOR:
    • autoclicker, aimassist, reach, velocity, killaura
    • scaffold, fly, speed, esp, xray, selfdestruct
    • bypass, exploit
    • Obfuscated names: a.class, b.class, single letters

HASH VERIFICATION
  What: Confirm file integrity via SHA256
  How: Get-FileHash or HashMyFiles
  How: Compare to official hash from Optifine.net, CurseForge, Modrinth
  RED FLAG: Mismatch = modified/tampered file

JAVAEDIT DETECTION
  What: Modified vanilla client .jar
  How: Hash the .jar in .minecraft/versions/{version}/
  How: Compare to official Mojang hash
  RED FLAG: Mismatch = modified vanilla client

UNLOADED MOD DETECTION
  What: Mod loaded then deleted/renamed after launch
  How: System Informer > javaw.exe > Memory > Strings
  How: Filter contains: full path to active mods folder
  How: Compare memory results to disk contents
  RED FLAG: Mod path in memory but file missing on disk

PREFETCH (C:\Windows\Prefetch)
  Tool: WinPrefetchView
  What: Shows every .exe ever run with timestamps
  Check if disabled:
    reg query "HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management\PrefetchParameters" /v EnablePrefetcher
    0 = disabled (RED FLAG)
  Check if service stopped:
    sc query sysmain
    RUNNING = good, STOPPED = RED FLAG
  View hidden prefetch files:
    dir /ah C:\Windows\Prefetch

BAM (Background Activity Moderator)
  Where: HKLM\SYSTEM\CurrentControlSet\Services\bam\State\UserSettings\{SID}
  What: Executed application paths + last execution time
  Tool: Deleted BAM Scanner (finds cleared entries)

PCA (Program Compatibility Assistant)
  Where: HKCU\Software\Microsoft\Windows NT\CurrentVersion\AppCompatFlags\Compatibility Assistant\Store
  What: Program paths PCA interacted with
  Tool: PcaSvcExecuted.exe (shows all EXEs ran + renamed/deleted)

CMD HISTORY
  If CMD still open:
    doskey /history
  PowerShell history path:
    (Get-PSReadLineOption).HistorySavePath
    Open that file path in explorer

EVENT IDS (Event Viewer)
  Security Log:
    1102 = Security log cleared (manual, admin required)
  System Log:
    104 = Other event log cleared
    4616 = System time changed
    7036 = Service state change (eventlog stopped/started)
  Application Log:
    3079 = USN Journal deleted (fsutil usn deletejournal)

USN JOURNAL
  Tool: USN.Journal.exe or JournalTrace
  CMD (fsutil):
    fsutil usn readjournal c: csv | findstr /i /C:"0x80000200" /C:"0x00001000" /C:"0x00002000" | findstr /i /C:".exe\"" /C:".pf\"" > filtered_journal.txt
  Reason codes:
    0x80000200 = Delete
    0x00001000 = Rename (old)
    0x00002000 = Rename (new)

ADS (Alternate Data Streams)
  CMD: dir /r
  PowerShell: Get-Item -Path .\file.txt -Stream *
  Tools: AlternateStreamView, Sysinternals Streams

RECUVA
  What: Deleted file recovery
  How: Quick scan > look for recently deleted .exe/.dll/.jar
  CRITICAL: DO NOT recover to original path

USN JOURNAL DELETION

EVENT LOG CLEARING
